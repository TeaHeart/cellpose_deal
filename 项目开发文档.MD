## 1.项目概述

### 1.1 项目背景

开发一款桌面应用，为科研人员提供从原始图片批量处理、交互式质检到结构化数据导出的完整本地化工作流。核心是整合细胞分割能力，并提供严谨的数据管理。

### 1.2 核心目标

安全、可追溯、结果即用。用户通过本工具，能从原始图片得到可直接用于统计分析的、带完整记录的纯净数据。

### 1.3 目标用户

需要处理大批量显微镜图片的实验室研究员、学生。

## 2.核心架构与数据模型

### 2.1 系统架构

```text
用户界面层
    │
    ▼
核心业务逻辑层 (数据包管理、编辑、导出)
    │
    ▼
【细胞分割处理器抽象层】 ←─ 关键解耦点
    │
    ▼
本地Cellpose实现
```

- **核心思想**：上层业务逻辑通过一个标准接口调用分割功能，与具体的Cellpose版本和调用方式解耦。

### 2.2 核心概念：数据包

- 为每张处理过的图片创建的唯一文件夹：`[图片名].segpkg`
- 内部结构：

```text
图片.segpkg/
├── 0_原始数据/image.tif
├── 1_处理结果/
│   ├── 原始遮罩.npy   # 分割处理器返回的数组，值=细胞ID
│   ├── 完整记录.csv  # 核心表，含所有细胞状态
│   └── 处理参数.json
├── 3_最终结果/       # （执行“最终合并”后生成）
│   ├── 最终遮罩.npy
│   └── 最终数据.csv
└── 清单.json
```

### 2.3 核心数据表：完整记录.csv

- **必须包含列**：
  - **测量列**：细胞ID, 面积, 直径, 中心X, 中心Y等。
  - **状态列**：状态(有效/已删除)、修改原因、来源(自动/手动)。
- **关键约束**：细胞ID必须与 `原始遮罩.npy` 中的整数标签严格一一对应。

### 2.4 核心交互原则：非破坏性编辑

- 所有人工操作不修改 `原始遮罩.npy`，只修改 `完整记录.csv` 中的状态列。

## 3.详细功能规格 (第一版)

### 3.1 模块一：项目初始化与处理

1. **文件夹扫描**
   - 递归扫描用户选择的根目录，找出支持的图片文件。

2. **参数配置面板**
   - 必填：像素尺度（微米/像素）。提供输入引导，尝试从TIFF元数据读取。
   - 核心参数：传递给分割处理器的参数字典（如模型类型、细胞直径等，带默认值）。
   - （可选）自动过滤规则：
     - 勾选框：`[ ]` 启用自动过滤
     - 可设置：直径范围、圆度、面积范围。

3. **安全预览机制**
   - 默认开启。点击“开始处理”后，随机选取2-3张图，通过分割处理器处理并弹窗展示结果。
   - 用户选择：返回修改参数或 继续处理全部。
   - 提供选项：`[ ]` 下次不再询问，直接处理。

4. **批量处理与自动过滤**
   - 通过分割处理器处理所有图片，将返回的遮罩数组保存为 `原始遮罩.npy`，并生成初始 `完整记录.csv`。
   - 若启用自动过滤，则根据规则修改CSV，标记不合格细胞。

### 3.2 模块二：交互式审查界面

1. **主界面布局**
   - 顶部：实时统计 **有效数(绿) / 已删数(红) / 总数(灰)**。数字可点击，用于筛选表格。
   - 左侧画布：显示原图+分割轮廓，标注细胞ID。
   - 右侧表格：显示 `完整记录.csv`，可排序。

2. **交互操作**
   - 双向联动高亮：点击画布轮廓或表格行，两侧同时高亮对应目标。
   - 删除细胞：选中后点“删除”，选择原因。更新表格状态，画布轮廓变灰。
   - 恢复细胞：在表格中直接修改 `状态列`，从已删除改回有效。

3. **辅助功能**
   - 图片导航（上一张/下一张）。
   - 项目管理界面中，可筛选 **全部、未审查、已审查、含删除项** 的图片。

### 3.3 模块三：结果生成与导出

1. **最终合并**
   - 用户可执行“生成最终结果”。
   - 逻辑：读取 `完整记录.csv`，提取有效细胞，重新组合生成 `3_最终结果/` 目录下的干净文件。
   - **注意**：直接覆盖旧文件，无版本回溯。

2. **一键导出**
   - 用户选择输出父目录。
   - 软件自动在该目录下创建带时间戳的新文件夹（如 `分析结果_20260401_1430`）。
   - 在此新文件夹内，镜像原始实验的目录结构。
   - **导出模式（单选）**：
     - **模式A：按图片导出**
       1. 在每个图片位置生成 `[图片名]_结果.csv`。内容为该图最终有效细胞数据，加元数据注释行。
     - **模式B：按样品文件夹合并导出**
       1. 分组逻辑：将同一直接父目录下的图片视为一个“样品组”。
       2. 输出：为每个这样的文件夹生成一个 `[文件夹名]_合并数据.csv`。
       3. 内容：是该文件夹下所有图片有效细胞数据的纵向合并，必须包含来源图片列。

### 3.4 模块四：高级工具

- **重新处理**：支持对单张/选中多张图片用新参数重跑。重处理后，覆盖原始数据，并重置所有编辑。

## 4.关键组件设计：细胞分割处理器

### 4.1 接口定义

定义一个名为 `SegmentationProcessor` 的接口（或抽象基类），核心方法如下：

```python
def process(self, image_path: str, params: dict) -> dict:
    """
    处理单张图片。
    输入: 图片路径，参数字典。
    返回: 标准结果字典。必须包含键：'success' (bool), 'mask_array' (np.ndarray 或 None), 'error_message' (str 或 None)。
    """
```

### 4.2 默认实现：本地Cellpose处理器

- **类名**：`LocalCellposeProcessor`
- **职责**：
  - 初始化时检查本地Cellpose环境。
  - 实现 `process` 方法，内部调用本地Cellpose库（如 `from cellpose import models`）。
  - 将Cellpose返回的 `masks` 数组，包装成接口要求的字典格式。
  - 处理并封装Cellpose运行中可能抛出的异常。
- **配置**：该处理器为第一版默认且唯一的实现。其初始化参数（如模型路径）可通过软件设置进行配置。

### 4.3 集成方式

- 核心业务逻辑（批处理引擎、重处理器）仅依赖 `SegmentationProcessor` 接口。
- 通过工厂模式或依赖注入获取处理器的实例。目前固定返回 `LocalCellposeProcessor` 的实例。

## 5.非功能性需求

- **性能**：图片加载、轮廓渲染需流畅。批量任务有进度显示。
- **健壮性**：单张图处理失败不应导致整个批处理任务崩溃。
- **错误处理**：对关键操作（重处理、最终合并）有明确警告；分割处理器的错误应被捕获并记录。
- **配置**：应允许高级用户配置本地Cellpose的Python环境路径（如果需要）。
