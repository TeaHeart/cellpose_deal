# Cellpose Deal 项目功能文档

## 一、项目概况

基于 Cellpose 的桌面端细胞/颗粒分割与质检工具，面向科研人员，提供从原始图片批量处理、交互式审查到结构化数据导出的完整本地化工作流。

**技术栈：** Python 3.11 + PySide6 (Qt GUI) + Cellpose 4.1.0

---

## 二、功能清单

### 1. 图像文件管理

#### 1.1 文件夹浏览（文件树）
- 通过 `QFileSystemModel` 展示所选目录的完整文件树
- 支持递归展开/折叠（单击节点切换）
- 自动展开到第 2 级深度，自适应列宽

#### 1.2 图片导航
- 菜单：**文件 → 上一张**（快捷键 `←`）/ **下一张**（快捷键 `→`）
- 在文件树中按顺序跳转至下一个 `.tif` 或 `.tiff` 图片

---

### 2. Cellpose 推理

#### 2.1 单张推理
- 菜单：**模型 → 推理选中**（快捷键 `Ctrl+R`）
- 对文件树中当前选中的图片执行分割

#### 2.2 批量推理
- 菜单：**模型 → 推理全部**（快捷键 `Ctrl+Shift+R`）
- 对目录下所有 `.tif`/`.tiff` 文件执行批量分割

#### 2.3 推理参数配置
界面左下角提供三个可调参数：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| 像素标尺 (px/um) | 像素转微米换算 | 18.5 |
| 颗粒预估直径 (px) | Cellpose `diameter` | 0（自动） |
| 迭代次数 | Cellpose `niter` | 0 |

- 默认值从 `config.yaml` 读取
- 推理后参数自动保存为同名 `.yaml`，下次打开图片时自动恢复

#### 2.4 后台推理与进度
- `InferenceWorker` 继承 `QThread`，后台执行不阻塞主界面
- 实时进度对话框显示当前进度 (i/total) 和正在处理的文件名
- 支持取消操作

#### 2.5 GPU 加速
- 使用 `models.CellposeModel(gpu=True)` 加载模型，支持 CUDA 推理
- 模型懒加载，首次使用时初始化

---

### 3. 图像可视化与交互

#### 3.1 原图显示
- 使用 `QGraphicsView` + `QGraphicsScene` 显示 TIFF 图片
- `fitInView` 自适应窗口，保持纵横比

#### 3.2 分割轮廓绘制
- 读取 `.npy` 掩码文件，用 OpenCV `findContours` 提取边界
- **绿色轮廓线** 显示每个细胞/颗粒
- **红色文字** 标注颗粒 ID 于质心位置

#### 3.3 轮廓点击选中与双向联动
- 点击图像中细胞轮廓 → 轮廓变 **红色高亮**
- 同时表格对应行自动选中并滚动到可见区域
- 点击表格某行 → 图像对应轮廓变红高亮
- 点击空白区域取消选中

#### 3.4 删除状态可视化
颜色编码：
| 颜色 | 状态 |
|------|------|
| 绿色 | 正常 |
| 红色 | 选中 |
| 黄色 | 已删除 |

#### 3.5 图像缩放与拖拽
- 鼠标滚轮缩放（1.25x 因子，鼠标位置为中心锚点）
- `ScrollHandDrag` 模式，可拖拽平移

#### 3.6 右键菜单删除/恢复
- 图像上右键点击细胞：
  - 未删除时 → **标记删除**
  - 已删除时 → **取消删除**
- 操作后自动更新颜色并保存 CSV

---

### 4. 数据表格与形态学分析

#### 4.1 形态学指标计算
基于 `skimage.measure.regionprops_table` 提取：
| 指标 | 说明 |
|------|------|
| 颗粒ID | 细胞标签编号 |
| 直径 | 等效直径（微米） |
| 圆度 | `4π·面积 / 周长²` |
| 长宽比 | 长轴 / 短轴 |
| 紧实度 | `solidity`（凸包填充度） |
| 面积 | 平方微米 |

#### 4.2 表格展示
- 自定义 `PandasTableModel` 绑定 pandas DataFrame
- 数值格式化为 4 位小数，数值型列右对齐
- 已删除行以 **黄色文字** 显示
- 单选模式

#### 4.3 表格右键菜单
- **标记删除 / 取消删除** — 切换删除状态
- **重新生成csv** — 从 `.npy` 文件重新生成 CSV 数据

---

### 5. 数据加载与持久化

#### 5.1 自动加载关联文件
选中图片时自动查找并加载同名文件：
- `{basename}.yaml` — 推理参数，自动恢复至界面
- `{basename}_seg.npy` — 分割掩码（受"加载遮罩"开关控制）
- `{basename}.csv` — 形态学数据，自动加载到表格

#### 5.2 遮罩加载开关
- 菜单：**模型 → 加载遮罩**（可勾选，默认开启）
- 关闭后选择图片时不加载 `.npy` 掩码

#### 5.3 CSV 自动保存
- 任何删除/恢复操作后自动保存 CSV（`utf-8-sig` 编码）

---

### 6. 数据导出

#### 6.1 导出为 Excel
- 菜单：**文件 → 导出为xlsx**（快捷键 `Ctrl+E`）
- 将目录下所有 `.csv` 合并为一个 `.xlsx`
- 每个 CSV 作为独立 Sheet，Sheet 名为相对路径（下划线连接）
- 导出时自动 **过滤已删除行** 和"已删除"列

---

### 7. 辅助工具

#### 7.1 UI 编译
- `python setup.py build_ui` 编译 `.ui` 和 `.qrc` 文件

#### 7.2 函数计时
- `@timer_decorator` 标记函数执行耗时

#### 7.3 文件扫描
- `list_images()` 递归扫描目录，支持自定义扩展名和深度限制
- 返回 `(文件路径, 父目录, 相对组名)` 三元组

## 五、待办事项

### 1. 依赖安装优化（高优先级）

**问题：** `pip install -r requirements.txt` 直接安装会失败。

- `torch==2.11.0+cu128` 和 `torchvision==0.26.0+cu128` 来自 PyTorch 官方 CUDA 源，需要指定 `--index-url` 才能下载
- 列出了 45 个包，其中大量是间接依赖（如 `MarkupSafe`、`six`、`filelock`），用户难以判断哪些是核心依赖
- CPU 推理极慢（单张图数十分钟），GPU 为刚需，但缺乏 CUDA 环境配置指引

**建议：**
- 精简 `requirements.txt` 仅保留直接依赖
- 提供安装文档，给出分步指引：先装 CUDA 版 `torch`，再装其余依赖
- 文档中标注 CUDA 驱动版本要求，提示 CPU 推理，推理速度较慢，而不是卡住了

---

### 2. 推理操作与模型参数重组（高优先级）

**问题：** 推理入口隐藏在菜单中

- 单张推理和批量推理都放在"模型"菜单项，路径深，不易发现
- 参数面板（左下角）和推理按钮割裂，用户调好参数后需要去菜单找操作入口

**建议：**
- 在参数面板添加 **"推理"** 按钮（单张）和 **"批量推理"** 按钮

---

### 3. 图片导航状态显示（高优先级）

**问题：** 导航功能藏在菜单中，用户不知道当前在第几张、总共多少张。

- 位置：[main_window.ui:208-223](src/ui/main_window.ui#L208-L223)
- 缺少可视的位置反馈

**建议：** 在底部状态栏添加导航控件，如 `← 第 3/15 张 →`，可直接点击箭头翻页，切换时实时更新。（保留快捷键）

---

### 4. 形态学指标批量筛选删除（高优先级）

**问题：** 目前删除操作只能手动逐个标记，无法根据形态学指标（圆度、长宽比、面积等）批量筛选并删除不符合条件的颗粒。

**建议：** 支持设置筛选条件（如"圆度 > 0.8"、"长宽比 < 2"），对当前或全部 CSV 批量标记删除不符合条件的行。支持撤销操作。

---

### 5. 撤销/重做功能（高优先级）

**问题：** 手动删除颗粒或批量筛选时可能误操作，无法恢复。

**建议：** 实现撤销/重做（Undo/Redo），覆盖单张图片的颗粒删除、批量筛选删除、图片标记等操作。

---

### 6. 遮罩缓存（中优先级）

**问题：** 每次切换图片都从磁盘 `np.load` 重新加载 `.npy` 文件，无内存缓存。

- 位置：[main_window.py:275-279](src/ui/main_window.py#L275-L279)
- 频繁切换图片时产生不必要的大文件 I/O

**建议：** 实现 LRU 内存缓存（如 `functools.lru_cache` 或 OrderedDict），缓存已加载的掩码数据。

---

### 7. 图片标记功能（中优先级）

**问题：** 批量推理后逐个审查时发现某些图片效果不佳（模糊、无细胞等），无法标记，之后也无法快速找到这些图片。

**建议：** 支持标记当前图片为"效果不佳"，标记后能在文件树中标识出来，方便后续筛选查找。

---

### 8. 缩略图查看遮罩（中优先级）

**问题：** 当前查看遮罩需要点击选中图片后在右侧大图查看器中加载，无法快速浏览整个目录下哪些图片已跑过遮罩、分割效果如何。

**建议：** 在文件树或独立面板中显示遮罩缩略图预览，无需选中即可查看分割结果概览。

---

### 9. 图片批量添加比例尺（中优先级）

**问题：** 导出结果图到报告或论文时需要比例尺，目前无法一键添加。

**建议：** 作为导出功能的一部分，支持为已处理的图片批量添加比例尺后导出，可预览并设置比例尺的位置、大小、颜色。

---

### 10. 推理中断恢复（低优先级）

**问题：** 批量推理中途失败或取消后，重新开始会全部重跑，浪费时间和算力。

**建议：** 推理时跳过已有 `.npy` 文件的图片，或提供续跑选项。

---

### 11. 统计图表（低优先级）

**问题：** 缺乏对整体数据分布的直观了解。

**建议：** 提供面积/圆度/直径等指标的分布直方图，帮助快速判断整体数据质量。
